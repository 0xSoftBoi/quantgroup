Mechanics

How it works

Four operations — all governed by the constant-product invariant.

⬡

Pool State

Each pair maps to a pool via keccak256(tokenA, tokenB). Stores reserveA, reserveB, totalShares.

＋

Add Liquidity

First deposit bootstraps at √(amountA·amountB). Subsequent deposits respect the existing ratio or are pro-rated down.

－

Remove Liquidity

Burns LP shares, returns a pro-rata slice of both reserves. No lock-up, no penalty.

⇄

Swap

Routes through the pool in either direction. Output from constant-product formula. 0.3% fee stays in pool for LPs.

Swapping tokenA for tokenB moves the pool state along the k-curve. Price impact increases as you move further along the curve.
