// Mechanics
⬡

Pool State

Each token pair maps to a pool via keccak256(tokenA, tokenB). Stores reserveA, reserveB, and totalShares.

+

Add Liquidity

First deposit bootstraps at the geometric mean √(amountA · amountB). Subsequent deposits must respect the existing ratio.

−

Remove Liquidity

Burns LP shares and returns a pro-rata slice of both reserves. No lock-up period.

⇄

Swap

Routes through the pool in either direction. Output computed from constant-product formula with 0.3% fee retained in pool.

Swap output formula
amountOut = ( amountIn × 997 × reserveOut )
          ─────────────────────────────────
          ( reserveIn × 1000 + amountIn × 997 )
